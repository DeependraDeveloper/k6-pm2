# PM2 (Process Manager)
- PM2 is a production process manager for Node.js applications. Its job is to keep your application alive, manage server resources, and scale your app across multiple CPU cores.

# What it does:

- Auto-Restart: If your app crashes due to an unhandled error, PM2 instantly restarts it.
- Cluster Mode: Runs multiple instances of your Node.js app on a single server to utilize all CPU cores.
- Zero-Downtime Reloads: Allows you to update your code without dropping active user connections.

# When you use it:
- When deploying your Node.js backend to a cloud server (like AWS, DigitalOcean, or an EC2 instance).

```
Install 
  npm i g pm2

Start at all cores of a system (cores=max)
 pm2 start <path_index.js> -i max 

 typecript
 pm2 start .\src\server.ts -i max --interpreter ts-node

Stop 
 pm2 stop all

monitor 
 pm2 monit

```

# k6 (Load Testing Tool)
- k6 is a developer-centric load testing tool built by Grafana. It simulates virtual users sending thousands of HTTP requests to your server to test performance.


# What it does:

- Simulates concurrent traffic (e.g., 500 users hitting your API at the exact same second).
- Measures metrics like latency (how fast the server responds), throughput (requests per second), and error rates.
- When you use it: During development or before major releases to verify that your system won't crash under pressure.

```
Start a script
  k6 run <script_path>
```

# Start an application with a custom name
pm2 start server.js --name "my-api"

# Start and scale across ALL available CPU cores (Cluster Mode)
pm2 start server.js -i max

# Scale an already running application up/down
pm2 scale my-api 4          # Set to 4 instances
pm2 scale my-api +2         # Add 2 more instances

# Restart commands
pm2 restart my-api          # Hard restart (brief downtime)
pm2 reload my-api           # Zero-downtime reload (restarts workers sequentially)

# Stop or Delete
pm2 stop my-api             # Stop the app without removing it from PM2 list
pm2 delete my-api           # Stop and remove from PM2 list
pm2 delete all              # Stop and remove ALL processes

# View list of all running processes, memory, and CPU usage
pm2 status

# Real-time interactive dashboard (CPU, Memory, Logs)
pm2 monit

# Inspect specific app details (paths, env vars, restart count, script args)
pm2 show my-api             # or pm2 show <id>

# Check system-level resources
pm2 sysmon                  # Displays OS host stats


# Tail real-time logs for all apps
pm2 logs

# Tail logs for a specific app or process ID
pm2 logs my-api
pm2 logs 0

# Limit log output length
pm2 logs --lines 50

# Clear all accumulated log files
pm2 flush

# Production Auto-Restart & Environment Management
Ensure your applications restart automatically if the underlying server reboots or crashes.

Bash
# Save current running process list
pm2 save

# Generate and configure system startup hook (boot on system restart)
pm2 startup

# Clear saved process snapshot
pm2 unstartup

# Pass specific environment variables
pm2 start server.js --env production
