# PM2 (Process Manager)
- PM2 is a production process manager for Node.js applications. Its job is to keep your application alive, manage server resources, and scale your app across multiple CPU cores.

# What it does:

- Auto-Restart: If your app crashes due to an unhandled error, PM2 instantly restarts it.
- Cluster Mode: Runs multiple instances of your Node.js app on a single server to utilize all CPU cores.
- Zero-Downtime Reloads: Allows you to update your code without dropping active user connections.

# When you use it:
- When deploying your Node.js backend to a cloud server (like AWS, DigitalOcean, or an EC2 instance).

```
Install 
  npm i g pm2

Start at all cores of a system (cores=max)
 pm2 start <path_index.js> -i max 

Stop 
 pm2 stop all

monitor 
 pm2 monit

```

# k6 (Load Testing Tool)
- k6 is a developer-centric load testing tool built by Grafana. It simulates virtual users sending thousands of HTTP requests to your server to test performance.


# What it does:

- Simulates concurrent traffic (e.g., 500 users hitting your API at the exact same second).
- Measures metrics like latency (how fast the server responds), throughput (requests per second), and error rates.
- When you use it: During development or before major releases to verify that your system won't crash under pressure.

```
Start a script
  k6 run <script_path>
```
